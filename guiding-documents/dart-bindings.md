# Dart/Flutter Bindings via flutter_rust_bridge

This document outlines the plan for creating Dart/Flutter bindings for RustGremlin using [flutter_rust_bridge](https://github.com/aspect-build/aspect_rules_swc/blob/main/docs/rules.md), enabling graph traversal capabilities in Flutter applications across all platforms.

## Goals

1. **Cross-platform support** - iOS, Android, macOS, Windows, Linux, and Web
2. **Full traversal API** exposed to Dart with Gremlin-style fluent interface
3. **Graph construction/mutation** APIs for building and modifying graphs
4. **Type-safe Dart API** - Auto-generated Dart types from Rust
5. **Async-first design** - Non-blocking graph operations
6. **mmap support** - Persistent storage on native platforms

## Comparison: flutter_rust_bridge vs Other Approaches

| Aspect | flutter_rust_bridge | dart:ffi (manual) | Platform Channels |
|--------|---------------------|-------------------|-------------------|
| Code generation | Automatic | Manual | Manual |
| Type safety | High (auto-generated) | Manual binding | JSON serialization |
| Performance | Native FFI | Native FFI | Slower (message passing) |
| Async support | Built-in | Manual | Built-in |
| Web support | Via wasm-bindgen | N/A | N/A |
| Memory management | Automatic | Manual | N/A |
| Learning curve | Medium | High | Low |

**Recommendation**: Use flutter_rust_bridge for automatic code generation, type safety, and cross-platform support.

### Why flutter_rust_bridge?

1. **Automatic code generation** - Generates Dart bindings from Rust function signatures
2. **Type mapping** - Rust structs become Dart classes automatically
3. **Memory safety** - Handles Rust ownership across FFI boundary
4. **Async support** - Rust async functions become Dart Futures
5. **Web support** - Generates WASM bindings for Flutter Web
6. **Multi-platform** - Single codebase works across all Flutter platforms

### Graph Construction

```dart

void main() async {
  // Create an in-memory graph
  final graph = await Graph.inMemory();
```

### Traversal API

```dart
import 'package:rustgremlin/rustgremlin.dart';

Future<void> main() async {
  final graph = await Graph.inMemory();
  // ... add vertices and edges ...

  // Get traversal source
  final g = graph.traversal();

  // Basic traversal - fluent API with async terminal steps
  final friends = await g
      .V()
      .has('person', 'name', 'Alice')
      .out('knows')
      .values(['name'])
      .toList();
  // => ['Bob']

  // With predicates
  final adults = await g
      .V()
      .hasLabel('person')
      .has('age', P.gte(18))
      .toList();

  // Path tracking
  final paths = await g
      .V()
      .has('name', 'Alice')
      .out('knows')
      .out('knows')
      .path()
      .toList();

  // Select with labels
  final connections = await g
      .V()
      .hasLabel('person')
      .as_('a')
      .out('knows')
      .as_('b')
      .select(['a', 'b'])
      .toList();
  // => [{'a': {...}, 'b': {...}}, ...]

  // Branching with union
  final neighbors = await g
      .V(ids: [aliceId])
      .union([
        Traversal().out('knows'),
        Traversal().in_('knows'),
      ])
      .toList();

  // Repeat with times
  final threeHops = await g
      .V()
      .has('name', 'Alice')
      .repeat(Traversal().out())
      .times(3)
      .toList();

  // Aggregation
  final avgAge = await g
      .V()
      .hasLabel('person')
      .values(['age'])
      .mean()
      .next();

  // Group by
  final byLabel = await g
      .V()
      .group()
      .byLabel()
      .byValue()
      .build()
      .next();
  // => {'person': [...], 'company': [...]}
}
```

```rust
use rustgremlin::value::{EdgeId, Value as RustValue, VertexId};

/// Dynamic value type for graph properties and traversal results.
#[frb(dart_metadata = ("freezed"))]
pub enum Value {
    Null,
    Bool(bool),
    Int(i64),
    Float(f64),
    String(String),
    List(Vec<Value>),
    Map(HashMap<String, Value>),
    Vertex(u64),
    Edge(u64),
}

impl From<RustValue> for Value {
    fn from(v: RustValue) -> Self {
        match v {
            RustValue::Null => Value::Null,
            RustValue::Bool(b) => Value::Bool(b),
            RustValue::Int(n) => Value::Int(n),
            RustValue::Float(f) => Value::Float(f),
            RustValue::String(s) => Value::String(s),
            RustValue::List(items) => Value::List(items.into_iter().map(Into::into).collect()),
            RustValue::Map(map) => {
                Value::Map(map.into_iter().map(|(k, v)| (k, v.into())).collect())
            }
            RustValue::Vertex(id) => Value::Vertex(id.0),
            RustValue::Edge(id) => Value::Edge(id.0),
        }
    }
}

impl From<Value> for RustValue {
    fn from(v: Value) -> Self {
        match v {
            Value::Null => RustValue::Null,
            Value::Bool(b) => RustValue::Bool(b),
            Value::Int(n) => RustValue::Int(n),
            Value::Float(f) => RustValue::Float(f),
            Value::String(s) => RustValue::String(s),
            Value::List(items) => RustValue::List(items.into_iter().map(Into::into).collect()),
            Value::Map(map) => {
```

use crate::api::traversal::TraversalSource;
use anyhow::{anyhow, Result};
use flutter_rust_bridge::frb;
use rustgremlin::prelude::*;
use rustgremlin::storage::InMemoryGraph;
use std::collections::HashMap;
use std::sync::Arc;

#[cfg(feature = "mmap")]
use rustgremlin::storage::MmapGraph;

/// A graph database instance.
///
/// Supports both in-memory and persistent (mmap) storage backends.
#[frb(opaque)]
pub struct Graph {
    inner: rustgremlin::Graph,
}

impl Graph {
    /// Create a new in-memory graph.
    ///
    /// Data is lost when the graph is dropped. Use for testing,
    /// temporary graphs, or when persistence isn't needed.
    #[frb(sync)]
    pub fn in_memory() -> Graph {
        Graph {
            inner: rustgremlin::Graph::in_memory(),
        }
    }

    /// Open or create a persistent graph at the given path.
    ///
    /// Uses memory-mapped storage for efficient persistence.
    /// Data survives application restarts.
    ///
    /// # Platform Support
    ///
    /// Available on iOS, Android, macOS, Windows, and Linux.
    /// Not available on Web (use `in_memory()` instead).
    #[cfg(feature = "mmap")]
    pub fn open(path: String) -> Result<Graph> {
        let storage = MmapGraph::open(&path)?;
        Ok(Graph {
            inner: rustgremlin::Graph::new(Arc::new(storage)),
        })
    }

    /// Add a vertex to the graph.
    ///
    /// Returns the ID of the newly created vertex.
    pub fn add_vertex(
        &self,
        label: String,
        properties: HashMap<String, Value>,
    ) -> Result<u64> {
        let props = dart_to_properties(properties);
        let storage = self.inner.storage();

        // For InMemoryGraph, we need mutable access
        // This is a simplified example - actual implementation needs interior mutability
        let inmem = storage
            .as_any()
            .downcast_ref::<InMemoryGraph>()
            .ok_or_else(|| anyhow!("Storage does not support add_vertex"))?;

        // Note: InMemoryGraph needs to use interior mutability (RwLock)
        // for this to work. See implementation notes.
        let id = inmem.add_vertex(&label, props);
        Ok(id.0)
    }

    /// Add an edge between two vertices.
    ///
    /// Returns the ID of the newly created edge.
    pub fn add_edge(
        &self,
        from: u64,
        to: u64,
        label: String,
        properties: HashMap<String, Value>,
    ) -> Result<u64> {
        let props = dart_to_properties(properties);
        let from_id = VertexId(from);
        let to_id = VertexId(to);

        let storage = self.inner.storage();
        let inmem = storage
            .as_any()
            .downcast_ref::<InMemoryGraph>()
            .ok_or_else(|| anyhow!("Storage does not support add_edge"))?;

        let id = inmem
            .add_edge(from_id, to_id, &label, props)
            .map_err(|e| anyhow!("{}", e))?;
        Ok(id.0)
    }

    /// Get the number of vertices in the graph.
    #[frb(sync)]
    pub fn vertex_count(&self) -> u64 {
        self.inner.storage().vertex_count()
    }

    /// Get the number of edges in the graph.
    #[frb(sync)]
    pub fn edge_count(&self) -> u64 {
        self.inner.storage().edge_count()
    }

    /// Create a traversal source for querying the graph.
    #[frb(sync)]
    pub fn traversal(&self) -> TraversalSource {
        TraversalSource::new(&self.inner)
    }

    /// Execute a GQL query against the graph.
    ///
    /// Supports MATCH, WHERE, RETURN, ORDER BY, LIMIT, and OFFSET clauses.
    pub fn gql(&self, query: String) -> Result<Vec<Value>> {
        let snapshot = self.inner.snapshot();
        let results = snapshot
            .gql(&query)
            .map_err(|e| anyhow!("GQL error: {}", e))?;
        Ok(results.into_iter().map(Into::into).collect())
    }

    ///
    /// Expected format:
    /// ```json
    /// {
    ///   "vertices": [
    ///     {"id": 1, "label": "person", "properties": {"name": "Alice"}}
    ///   ],
    ///   "edges": [
    ///     {"id": 1, "label": "knows", "from": 1, "to": 2, "properties": {}}

### src/api/traversal.rs

```rust
use crate::api::predicate::Predicate;
use crate::api::value::Value;
use anyhow::Result;
use flutter_rust_bridge::frb;
use rustgremlin::prelude::*;
use rustgremlin::traversal::{self, p};

/// Entry point for graph traversals.
#[frb(opaque)]
pub struct TraversalSource {
    graph: rustgremlin::Graph,
}

impl TraversalSource {
    pub(crate) fn new(graph: &rustgremlin::Graph) -> Self {
        Self {
            graph: graph.clone(),
        }
    }

    /// Start traversal from all vertices or specific IDs.
    #[frb(sync)]
    pub fn v(&self, ids: Option<Vec<u64>>) -> Traversal {
        Traversal::from_vertices(&self.graph, ids)
    }

    /// Start traversal from all edges or specific IDs.
    #[frb(sync)]
    pub fn e(&self, ids: Option<Vec<u64>>) -> Traversal {
        Traversal::from_edges(&self.graph, ids)
    }

    /// Inject arbitrary values into the traversal.
    #[frb(sync)]
    pub fn inject(&self, values: Vec<Value>) -> Traversal {
        Traversal::from_inject(&self.graph, values)
    }
}

/// A graph traversal that accumulates steps and executes on terminal call.
#[frb(opaque)]
pub struct Traversal {
    graph: rustgremlin::Graph,
    steps: Vec<TraversalStep>,
    source: TraversalSourceType,
}

#[derive(Clone)]
enum TraversalSourceType {
    AllVertices,
    Vertices(Vec<u64>),
    AllEdges,
    Edges(Vec<u64>),
    Inject(Vec<Value>),
}

#[derive(Clone)]
enum TraversalStep {
    // Navigation
    Out(Vec<String>),
    In(Vec<String>),
    Both(Vec<String>),
    OutE(Vec<String>),
    InE(Vec<String>),
    BothE(Vec<String>),
    OutV,
    InV,
    BothV,
    OtherV,

    // Filter
    HasLabel(Vec<String>),
    Has(String),
    HasValue(String, Value),
    HasWhere(String, Predicate),
    HasId(Vec<u64>),
    HasNot(String),
    Is(Predicate),
    Dedup,
    Limit(usize),
    Skip(usize),
    Range(usize, usize),
    SimplePath,
    CyclicPath,
    Where(Box<Traversal>),
    Not(Box<Traversal>),
    And(Vec<Traversal>),
    Or(Vec<Traversal>),

    // Transform
    Values(Vec<String>),
    Id,
    Label,
    ValueMap(Vec<String>, bool), // keys, with_tokens
    ElementMap(Vec<String>),
    Path,
    Unfold,
    Constant(Value),
    As(String),
    Select(Vec<String>),

    // Aggregate
    Count,
    Sum,
    Mean,
    Min,
    Max,
    Fold,
    Group(GroupConfig),
    GroupCount(GroupCountConfig),

    // Branch
    Union(Vec<Traversal>),
    Coalesce(Vec<Traversal>),
    Optional(Box<Traversal>),
    Choose(Box<Traversal>, Box<Traversal>, Option<Box<Traversal>>),
    Local(Box<Traversal>),

    // Repeat
    Repeat(Box<Traversal>, RepeatConfig),

    // Order
    Order(Vec<OrderKey>),
    Project(Vec<String>, Vec<ProjectBy>),
}

#[derive(Clone)]
struct GroupConfig {
    by_key: GroupKey,
    by_value: GroupValue,
}

#[derive(Clone)]
enum GroupKey {
    Label,
    Property(String),
    Traversal(Box<Traversal>),
}

#[derive(Clone)]
enum GroupValue {
    Value,
    Property(String),
    Count,
    Traversal(Box<Traversal>),
}

#[derive(Clone)]
struct GroupCountConfig {
    by_key: GroupKey,
}

#[derive(Clone)]
struct RepeatConfig {
    times: Option<usize>,
    until: Option<Box<Traversal>>,
    emit: bool,
}

#[derive(Clone)]
enum OrderKey {
    Natural(bool), // ascending
    Property(String, bool),
    Traversal(Box<Traversal>, bool),
}

#[derive(Clone)]
enum ProjectBy {
    Property(String),
    Traversal(Box<Traversal>),
}

impl Traversal {
    fn from_vertices(graph: &rustgremlin::Graph, ids: Option<Vec<u64>>) -> Self {
        Self {
            graph: graph.clone(),
            steps: Vec::new(),
            source: ids
                .map(TraversalSourceType::Vertices)
                .unwrap_or(TraversalSourceType::AllVertices),
        }
    }

    fn from_edges(graph: &rustgremlin::Graph, ids: Option<Vec<u64>>) -> Self {
        Self {
            graph: graph.clone(),
            steps: Vec::new(),
            source: ids
                .map(TraversalSourceType::Edges)
                .unwrap_or(TraversalSourceType::AllEdges),
        }
    }

    fn from_inject(graph: &rustgremlin::Graph, values: Vec<Value>) -> Self {
        Self {
            graph: graph.clone(),
            steps: Vec::new(),
            source: TraversalSourceType::Inject(values),
        }
    }

    // -------------------------------------------------------------------------
    // Navigation Steps
    // -------------------------------------------------------------------------

    /// Traverse to outgoing adjacent vertices.
    #[frb(sync)]
    pub fn out(&mut self, labels: Option<Vec<String>>) -> &mut Self {
        self.steps.push(TraversalStep::Out(labels.unwrap_or_default()));
        self
    }

    /// Traverse to incoming adjacent vertices.
    #[frb(sync)]
    pub fn in_(&mut self, labels: Option<Vec<String>>) -> &mut Self {
        self.steps.push(TraversalStep::In(labels.unwrap_or_default()));
        self
    }

    /// Traverse to adjacent vertices in both directions.
    #[frb(sync)]
    pub fn both(&mut self, labels: Option<Vec<String>>) -> &mut Self {
        self.steps.push(TraversalStep::Both(labels.unwrap_or_default()));
        self
    }

    /// Traverse to outgoing edges.
    #[frb(sync)]
    pub fn out_e(&mut self, labels: Option<Vec<String>>) -> &mut Self {
        self.steps.push(TraversalStep::OutE(labels.unwrap_or_default()));
        self
    }

    /// Traverse to incoming edges.
    #[frb(sync)]
    pub fn in_e(&mut self, labels: Option<Vec<String>>) -> &mut Self {

        self.steps.push(TraversalStep::Optional(Box::new(traversal)));
        self
    }

    // -------------------------------------------------------------------------
    // Repeat Steps
    // -------------------------------------------------------------------------

    /// Repeat a traversal.
    #[frb(sync)]
    pub fn repeat(&mut self, traversal: Traversal) -> &mut Self {
        self.steps.push(TraversalStep::Repeat(
            Box::new(traversal),
            RepeatConfig {
                times: None,
                until: None,
                emit: false,
            },
        ));
        self
    }

    /// Repeat a fixed number of times.
    #[frb(sync)]
    pub fn times(&mut self, n: usize) -> &mut Self {
        if let Some(TraversalStep::Repeat(_, config)) = self.steps.last_mut() {
            config.times = Some(n);
        }
        self
    }

    /// Repeat until condition is met.
    #[frb(sync)]
    pub fn until(&mut self, traversal: Traversal) -> &mut Self {
        if let Some(TraversalStep::Repeat(_, config)) = self.steps.last_mut() {
            config.until = Some(Box::new(traversal));
        }
        self
    }

    /// Emit values during repeat.
    #[frb(sync)]
    pub fn emit(&mut self) -> &mut Self {
        if let Some(TraversalStep::Repeat(_, config)) = self.steps.last_mut() {
            config.emit = true;
        }
        self
    }

    // -------------------------------------------------------------------------
    // Terminal Steps
    // -------------------------------------------------------------------------

    /// Execute and return all results.
    pub fn to_list(&self) -> Result<Vec<Value>> {
        let results = self.execute()?;
        Ok(results.into_iter().map(Into::into).collect())
    }

    /// Execute and return first result.
    pub fn next(&self) -> Result<Option<Value>> {
        let mut results = self.execute()?;
        Ok(results.pop().map(Into::into))
    }

    /// Execute and return exactly one result.
    pub fn one(&self) -> Result<Value> {
        let results = self.execute()?;
        if results.len() != 1 {
            anyhow::bail!("Expected exactly 1 result, got {}", results.len());
        }
        Ok(results.into_iter().next().unwrap().into())
    }

    /// Execute and check if any results exist.
    pub fn has_next(&self) -> Result<bool> {
        let results = self.execute()?;
        Ok(!results.is_empty())
    }

    /// Execute and return count.
    pub fn count(&self) -> Result<u64> {
        // Add count step and execute
        let mut clone = self.clone();
        clone.steps.push(TraversalStep::Count);
        let results = clone.execute()?;
        match results.first() {
            Some(rustgremlin::Value::Int(n)) => Ok(*n as u64),
            _ => Ok(0),
        }
    }

    // -------------------------------------------------------------------------
    // Internal Execution
    // -------------------------------------------------------------------------

    fn execute(&self) -> Result<Vec<rustgremlin::Value>> {
```

## Platform-Specific Notes

### Android

- Minimum SDK: 21 (Android 5.0)
- Uses `cargo-ndk` for cross-compilation
- Native libraries placed in `jniLibs/` folder
- Supports: armeabi-v7a, arm64-v8a, x86_64

### iOS

- Minimum iOS: 12.0
- Uses `lipo` to create universal binary
- Static library linked into app

### macOS

- Minimum macOS: 10.14
- Supports: Apple Silicon (arm64) and Intel (x86_64)
- Universal binary for both architectures

### Linux

- Requires glibc 2.17+
- Supports: x86_64, aarch64

### Windows

- Minimum: Windows 10
- Supports: x86_64

### Web

- Uses wasm-bindgen under the hood
- No mmap support (in-memory only)
- Single-threaded execution

## Limitations

1. **Web limitations**:
   - No persistent storage (mmap not available)
   - Single-threaded execution
   - Memory limited by browser

2. **Platform differences**:
   - Path handling differs between platforms
   - File permissions may affect mmap on some platforms

3. **Current implementation gaps**:
   - Some advanced traversal steps not yet exposed
   - Async streaming not implemented
   - GraphQL integration pending

## Future Enhancements

1. **Streaming results** - Return `Stream<Value>` for large result sets
2. **Background isolates** - Run traversals in separate isolates
3. **IndexedDB fallback** - Persistence for web platform
4. **GraphQL integration** - Generate GraphQL schema from graph
5. **Riverpod/Bloc integration** - State management helpers
6. **Graph visualization widget** - Flutter widget for rendering graphs
